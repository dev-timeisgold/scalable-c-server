# 확장 가능한 C 서버 프로젝트 (Scalable C Server)

기본적인 TCP 에코 서버에서 시작하여, 대규모 트래픽 처리가 가능한 이벤트 기반(Event-Driven) 아키텍처로 점진적으로 발전해 나가는 고성능 네트워크 서버 구축 프로젝트입니다.

## 진행 단계
- [x] Phase 1: 기본 TCP 에코 서버 (단일 스레드 구조 / 블로킹 I/O)
- [x] Phase 2: 멀티 스레드 서버 (연결당 스레드 구조 / 동시성 확보)
- [ ] Phase 3: I/O 멀티플렉싱 서버 (epoll 기반 대용량 처리)

---

## Phase 1: 기본 TCP 에코 서버 (단일 스레드)

### 구현 핵심 (시스템 콜 파이프라인)
리눅스 커널의 네트워크 시스템 콜을 사용하여 서버의 가장 기초적인 파이프라인을 구축했습니다.
1. `socket()`: IPv4 주소 체계와 TCP 프로토콜을 사용하는 가상 파일 디스크립터(FD) 생성.
2. `setsockopt()`: 서버 비정상 종료 후 재부팅 시 포트가 잠기는 현상을 방지하기 위해 `SO_REUSEADDR` 옵션 적용.
3. `bind()`: 서버 소켓에 특정 포트(8080)와 네트워크 인터페이스 주소를 결합.
4. `listen()`: 커널 영역에 클라이언트 연결을 대기시킬 수 있는 수신 대기열(Listen Queue) 확보.
5. `accept()`: 대기열에서 클라이언트를 꺼내와 통신을 위한 새로운 소켓 FD를 발급하고 1:1 연결 수립.
6. `read()` / `send()`: 수립된 소켓을 통해 클라이언트의 메시지를 수신하고 그대로 반사(Echo).
7. `close()`: 통신 완료 후 리소스(FD) 반납.

### 아키텍처의 한계 (Phase 2 도입 배경)
* **치명적인 블로킹(Blocking) 문제:** 단일 스레드 환경이므로, `accept()`로 손님을 받은 후 `read()` 단계에서 클라이언트가 메시지를 보낼 때까지 서버 전체가 멈춰버리는 현상 발생.
* **결과:** 한 명의 클라이언트가 연결을 유지한 채 아무 동작을 하지 않으면, 대기열에 있는 다른 모든 클라이언트는 서버에 진입할 수 없음.

## Phase 2: 멀티 스레드 아키텍처 (현재 상태)

### 아키텍처 개요
Phase 1의 블로킹 문제를 해결하기 위해 `연결당 스레드(Thread-per-connection)` 모델을 구현하였습니다.
1. **메인 스레드 (`accept` 담당)**: 오직 서버의 정문(Listening Socket)을 모니터링하며 연결 수락에만 전념.
2. **워커 스레드 (`read`/`send`/`close` 담당)**: 메인 스레드가 `pthread_create()`를 통해 동적으로 생성하며, 단일 클라이언트 연결의 통신 수명 주기를 전담 처리.

### 주요 기술 상세
* **동시성(Concurrency) 확보**: POSIX 스레드(`pthreads`)를 통해 멈춤 현상(Non-Blocking) 없이 다수의 클라이언트 동시 응대 가능.
* **메모리 데이터 경합(Data Race) 방지**: 클라이언트 소켓 파일 디스크립터를 워커 스레드로 넘길 때 주소값을 직접 넘기지 않고, 독립된 힙 메모리(`malloc`)를 할당하여 전달. 이를 통해 메인 스레드의 변수 덮어쓰기로 인한 소켓 꼬임 현상 원천 차단.
* **스레드 생명주기 관리 (`detach` vs `join`)**: 네트워크 서버의 특성상 메인 스레드가 워커 스레드의 종료를 기다려줄(`join`) 수 없으므로, 스레드를 분리 상태(`pthread_detach`)로 생성하여 업무 종료 즉시 메모리 리소스가 자동 반환되도록 설계.

### 아키텍처의 한계 (Phase 3 도입 배경)
* **C10K 문제 (1만 명 동시 접속 문제):** 클라이언트 수가 기하급수적으로 늘어날 경우, 스레드 생성 개수도 동일하게 증가.
* **결과:** 스레드당 할당되는 스택 메모리(보통 8MB)로 인해 서버 RAM이 고갈되며, 수많은 스레드를 교체하며 실행하는 컨텍스트 스위칭(Context Switching) 과정에서 CPU 오버헤드가 극심해짐.

## 빌드 및 관리 명령어 (Makefile)
이 프로젝트는 `Makefile`을 제공하여 컴파일, 실행, 리소스 정리 프로세스를 자동화합니다.

```bash
# 1. 서버 컴파일 (코드 빌드)
make

# 2. 서버 실행 (포트 8080 개방)
make run

# 3. 빌드 잔여물 삭제 (기존 실행 파일 철거 및 초기화)
make clean
